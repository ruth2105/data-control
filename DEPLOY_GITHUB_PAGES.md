## Step-by-Step Deployment

### Step 1: Create GitHub Account (if you don't have one)
1. Go to https://github.com
2. Click **Sign up**
3. Enter email, password, username
4. Verify email address

### Step 2: Create New Repository
1. Log into GitHub
2. Click the **+** icon (top right) → **New repository**
3. Repository name: `hospital-list` (or any name you want)
4. Choose **Public** or **Private** (both work with Pages)
5. ✅ Check "Add a README file"
6. Click **Create repository**

### Step 3: Upload Your Files
1. In your new repository, click **Add file** → **Upload files**
2. Drag and drop these files:
   - `index.html` ⭐ MOST IMPORTANT
   - `appsscript.gs` (optional - for reference)
   - All `.md` documentation files (optional)
3. Scroll down and click **Commit changes**

### Step 4: Enable GitHub Pages
1. In your repository, click **Settings** (top menu)
2. Scroll down left sidebar → Click **Pages**
3. Under "Source":
   - Branch: Select **main** (or **master**)
   - Folder: Select **/ (root)**
4. Click **Save**
5. Wait 1-2 minutes

### Step 5: Get Your Live URL
1. Refresh the Pages settings page
2. You'll see: **"Your site is live at https://yourusername.github.io/hospital-list"**
3. Click the URL to open your app!

## 🔒 Security Notes

### Google Sheets Connection:
Your app connects to Google Sheets via the Apps Script URL (`SCRIPT_URL`). This remains secure because:
- ✅ Only authenticated users can access the sheet
- ✅ The script URL is public but data access requires permissions
- ✅ All communication is over HTTPS

### If You Want Extra Security:
You can make the GitHub repository **Private**:
1. Go to repository Settings
2. Scroll to bottom → **Danger Zone**
3. Click **Change visibility** → **Make private**
4. GitHub Pages still works with private repos!

## 🆘 Troubleshooting

### "404 Page Not Found"
- Wait 2-5 minutes after enabling Pages
- Make sure file is named exactly `index.html` (lowercase)
- Check that Pages is enabled in Settings

### "Changes Not Showing"
- Wait 1-2 minutes after committing
- Clear browser cache (Ctrl+Shift+R or Cmd+Shift+R)
- Try opening in incognito/private window

### "Data Not Saving"
- Make sure your Google Apps Script is deployed
- Check that `SCRIPT_URL` in `index.html` is correct
- Test the Google Sheets connection

## 💡 Pro Tips

### Custom Domain (Optional)
Want a custom URL like `hospital.yourcompany.com`?
1. Buy a domain (from Namecheap, GoDaddy, etc.)
2. In GitHub Pages settings, enter your custom domain
3. Update DNS records at your domain provider
4. FREE with GitHub Pages!

### Automatic Backups
- Every change you commit is automatically backed up
- You can view history: Click on `index.html` → **History**
- Restore old versions anytime

### Collaboration
Want other developers to help?
1. Go to repository Settings → **Collaborators**
2. Click **Add people**
3. Enter their GitHub username
4. They can now edit the code!
